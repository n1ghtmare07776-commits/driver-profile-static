# Agent 约束（已合并）

完整且与当前页面同步的交接说明见 [AGENT_HANDOFF_统一说明.md](AGENT_HANDOFF_统一说明.md)。本文件保留为自动识别入口；如与统一说明冲突，以统一说明为准。

你正在复刻一个内部客服使用的“司机画像”网站。请严格按以下约束执行。

## 总目标

在新项目中实现一个可运行网站，功能和外观参考当前“司机画像”最新版。本次需要实现建议策略模块，但不需要话术模板。当前生产策略按 case 均值触发，但必须配置驱动，不得写死在页面渲染代码里。

## 必须保留的功能

- 页面标题必须是 `司机画像`。
- 左侧为筛选区和司机列表，右侧为司机画像详情。
- 支持司机 ID 查询。
- 支持城市多选筛选。
- 支持公司多选筛选。
- 支持产品线（二级）多选筛选。
- 支持数据日期筛选。
- 支持高级筛选的最小值和最大值。
- 筛选组之间必须使用 AND。
- 城市、公司、产品线多选组内部使用 OR。
- 城市、公司、产品线下拉框默认收起，点击输入框后展开。
- 多选项必须支持搜索、点击加入、再次点击移除，并显示已选标签和右侧对勾。
- 支持下载当前筛选名单 CSV。
- CSV 至少包含 `司机id` 和 `综合画像` 两列。
- 详情页必须显示数据日期或快照日期。
- 详情页必须显示 `建议策略` 模块，位置在 `综合画像` 之后、`司机资料` 之前。
- 建议策略必须配置驱动，不允许把阈值数字、触发规则和建议文案散落写死在页面渲染代码里。
- 当前生产建议策略只使用 `case_mean`：大部分司机字段值高于死亡 case 文件中对应字段的 case 均值才触发展示；`min_sleep_duration` 等反向指标可通过配置使用低于 case 均值触发。
- 不使用非 case 均值口径的人工阈值、兜底阈值或宽泛城市/健康标签触发；后续更换阈值算法时，通过配置和独立计算层替换。
- 建议策略缺少司机指标、参考指标、有效样本或阈值时，不触发对应策略，并保留可排查说明。
- 无策略命中时必须显示 `常规关怀`。
- 建议策略文案必须委婉、关怀、可解释、可操作，不做医学诊断，不做事故因果判断。
- 由于 CDN 一键部署包大小上限为 800MB，生产发布包不得持续累计历史数据库或历史司机明细。
- 后续每次接收新一天数据时，默认只保留上传当天向前含当天的最新 7 个自然日数据用于 CDN 发布和页面查询。
- CDN 数据构建或上传脚本必须执行 7 天窗口过滤，并输出可追溯的数据窗口信息。
- 如果最近 7 天静态产物超过 CDN 800MB 上限，脚本必须先裁剪非页面必需字段并使用紧凑 JSON；仍超过时，按日期从旧到新剔除，只保留 800MB 内可容纳的最近日期，且至少尝试保留最新 1 天。
- 因 800MB 上限剔除日期时，必须在 manifest 或元信息中记录 `actual_dates`、`dropped_dates`、`drop_reason`、`package_size_mb` 和 `package_size_limit_mb`。
- CDN 静态版改用按日独立的稳定静态版方案：首页先加载 `manifest.json`、`filter-options.json` 和 `drivers.json` 日期索引；每天司机明细必须写入独立文件 `daily/drivers-YYYY-MM-DD.json`，切换数据日期时加载对应日期文件。
- `drivers.json` 只能作为日期索引和文件清单，不得把多天司机明细合并成一个会被每日更新覆盖的大文件。
- 每次接收新一天数据时，脚本必须保留 7 天窗口内已有日期文件，并只覆盖本次输入对应日期的 `daily/drivers-YYYY-MM-DD.json`；禁止因为只上传 07-07 单日数据就删除 07-03 等仍在窗口内的日期文件。
- 每个 `daily/drivers-YYYY-MM-DD.json` 只能包含该自然日的查询、筛选、列表、详情、综合画像和建议策略展示所需数据，不得包含其他日期生产司机明细或页面不需要的全量原始字段。

## 可以去掉的内容

- 话术模板模块。
- 语料库文件。
- 本地 SQLite 重建按钮。
- 旧项目中的全部字段明细大表。
- 写死在页面渲染代码里的建议策略。
- 不可配置、不可替换、只嵌在 UI 里的建议策略逻辑。
- 持续追加历史数据的本地数据库或 CDN 数据包。

## 页面要求

- 页面语言必须是中文。
- 使用白色背景，整体清爽、可读。
- 保持桌面端布局，优先适配客服电脑使用。
- 不要做营销落地页，不要做大 hero。
- 不要把页面改成深色主题。
- 不要让筛选栏一打开就被长下拉框撑开。
- 不要使用会遮挡内容的弹层。

## 数据和安全

- 不要在代码里写真实司机数据。
- 示例数据只能使用脱敏或假数据。
- 不要把真实 driver_id、健康指标或完整 profile 写进文档样例。
- 如果最终使用静态 JSON，必须清楚标注静态 JSON 包含可查询数据。
- 生产数据应通过每日数据处理脚本生成 CDN 静态 JSON 或 API 数据源；代码仓库和文档样例不得提交真实司机明细。
- 每日数据处理必须按上传日默认截取最新 7 天，并在 manifest 或元信息中记录 `generated_at`、`upload_date`、`window_start`、`window_end`、`row_count`。
- 如果实际保留日期少于默认 7 天，前端和文档必须以 `manifest.actual_dates` 为准，并可解释 `dropped_dates` 的原因。
- `generated_at` 必须由构建或上传脚本在生成静态站点产物时写入当前实时时间，不能手工写死。
- 前端必须展示当前数据日期或快照日期；如果使用 CDN 静态 JSON，也应能展示当前数据窗口或最新快照日期。
- CDN 静态版左侧的 `静态站点生成时间` 必须展示 `manifest.generated_at` 格式化后的时间，格式建议为 `YYYY/MM/DD HH:mm`。

## 工程要求

- 可以选择任意简单栈，但必须能本地启动。
- 如果做静态版，入口应是 `index.html`，数据从本地 JSON 文件读取。
- CDN 静态版入口是 `dist/index.html`，不使用也不要求 `index-inline.html`；不得把全部司机数据内联进单个 HTML。
- 如果做后端版，应提供清晰的 API。
- 如果使用 CDN 静态版，必须提供数据构建/上传脚本说明：输入每日数据，过滤最新 7 天，生成稳定静态版 `drivers.json` 日期索引、`daily/drivers-YYYY-MM-DD.json` 按日明细、`filter-options.json`、策略阈值文件和 manifest，并把构建实时时间写入 `manifest.generated_at`，再上传到 CDN。
- 建议策略必须有独立配置文件和独立计算层，页面只消费计算后的策略结果。
- 必须提供启动命令、构建命令和验收步骤。
- 必须提供至少一份样例数据，能完成查询、筛选和下载验收。

## 禁止事项

- 禁止只做 UI 截图或不可交互页面。
- 禁止只实现司机 ID 查询而丢掉组合筛选。
- 禁止只上传或只生成单个 HTML 后丢掉数据文件。
- 禁止把导出改成全字段泄露导出。
- 禁止使用旧版深色原型替代当前白色版本。
- 禁止删除建议策略模块。
- 禁止把建议策略实现成固定文案或固定阈值数字。
- 禁止把超过默认最新 7 天窗口的生产司机明细上传到 CDN，除非用户明确改口并记录原因。
- 禁止把超过最近 7 天窗口的数据、页面不需要的全量原始字段或多天司机明细塞进 `drivers.json`。
