# 页面外观参考

请参考 `reference/current-cdn-frontend/` 和 `reference/current-local-frontend/` 中的现有 HTML/CSS/JS。

本次复刻不要求逐行复制代码，但视觉和交互应该接近当前白色版本。

## 整体布局

```text
┌──────────────────────────────────────────────────────────────┐
│ 左侧筛选栏 300px 左右            │ 右侧司机画像详情区           │
│                                  │                            │
│ 司机画像                         │ 初始提示 / 司机详情           │
│ 数据量和日期                     │                            │
│                                  │ 顶部司机 ID 卡片              │
│ 司机ID输入                       │                            │
│ 城市多选                         │ 1 综合画像                    │
│ 公司多选                         │                            │
│ 产品线多选                       │ 2 建议策略                    │
│ 日期筛选                         │   基础资料卡片                │
│ 高级筛选                         │   出车/在线指标卡片            │
│ 查询 / 重置                      │   疲劳相关卡片                │
│ 下载当前筛选名单                  │   健康相关卡片                │
│ 司机结果列表                     │                            │
└──────────────────────────────────────────────────────────────┘
```

## 视觉风格

- 背景：浅灰 `#f5f7fb`。
- 卡片背景：白色。
- 主色：蓝色，类似 `#1467b3`。
- 边框：浅灰蓝，类似 `#d7e0ea`。
- 字体：系统中文字体，优先 `PingFang SC`、`Microsoft YaHei`。
- 卡片圆角：6-8px。
- 页面是工具型产品，不是营销页。

## 左侧筛选栏

左侧宽度约 300px，固定在页面左边。包含：

- 标题 `司机画像`。
- 数据摘要，例如 `5000位司机 · 2026-06-30`。
- 筛选表单。
- 查询结果列表。

多选组件要求：

- 初始只显示输入框，不展开候选列表。
- 点击输入框后展开下拉候选。
- 输入关键词后过滤候选。
- 点击候选后添加成标签。
- 已选项右侧显示对勾。
- 再次点击候选或标签可以移除。
- 城市可优先展示常用城市：北京、上海、广州、深圳、杭州、成都、重庆、武汉、南京、天津、西安、苏州。

## 右侧详情区

初始状态显示：

```text
请选择或输入一个司机ID。
```

司机详情顶部卡片：

- 左侧显示 `司机画像` 小标题。
- 主标题显示 `司机ID xxxxxx`。
- 下方展示标签，如城市、产品线、疲劳/高龄等问题标签。
- 右侧可显示分数排名或风险排名，如果没有数据则显示 `暂无数据`。

主要内容：

1. 综合画像
2. 建议策略
3. 司机资料

建议策略卡片展示：

```text
策略标题
依据：xxx
建议：xxx
```

策略文案应委婉、关怀、可解释，不做医学诊断或事故因果判断。

本次不要展示：

- 话术模板

司机资料建议用 2-3 列卡片网格展示，按字段分组。

## 空值规则

- 空值、null、空字符串显示 `暂无数据`。
- 数字 0 必须显示 `0`，不能误判为空。
