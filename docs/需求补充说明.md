# 需求补充说明

## 背景

这个网站给同事使用。使用者在联系司机前，可以通过司机 ID 或筛选条件查看司机画像，从而更快了解司机的基本情况、工作强度和疲劳相关数据。

本次不是继续维护旧项目，而是在新文件夹里复刻核心能力。重点是页面可用、查询可用、筛选可用、建议策略可用、下载可用。

## 用户

- 使用者是公司内部同事。
- 用户知道司机 ID，也可能按城市、公司、产品线、日期和风险/疲劳字段筛选名单。
- 用户界面必须使用中文。

## 当前核心页面

页面左侧是筛选和司机列表，右侧是司机档案。

左侧包含：

- 司机 ID 输入。
- 城市多选。
- 公司多选。
- 产品线（二级）多选。
- 数据日期。
- 高级筛选。
- 查询按钮。
- 重置按钮。
- 下载当前筛选名单按钮。
- 查询结果列表。

右侧包含：

- 初始空状态。
- 查询中状态。
- 未找到状态。
- 司机详情。
- 综合画像。
- 建议策略。
- 司机资料分组。

## 功能口径

### 查询

- 输入纯数字司机 ID 后可以查询。
- 不存在的司机 ID 要显示未找到。
- 无效格式要提示，不要崩溃。
- 页面初始状态不要展示默认全量司机列表。

### 组合筛选

- 支持城市、公司、产品线、日期、高级筛选组合。
- 不同筛选组之间必须是 AND。
- 城市多选组内是 OR。
- 公司多选组内是 OR。
- 产品线多选组内是 OR。

示例：

```text
城市 = 北京市 或 上海市
公司 = A公司 或 B公司
产品线 = 快车
年龄 >= 55
连续出车天数 >= 3
```

含义是：

```text
(北京 或 上海) AND (A公司 或 B公司) AND 快车 AND 年龄>=55 AND 连续出车天数>=3
```

### 高级筛选

必须支持以下数值字段的最小值和最大值：

| 中文名 | 字段名 |
|---|---|
| 年龄 | `age` |
| 连续出车天数 | `consecutive_days` |
| 服务时长 | `server_dur_hour` |
| 夜间出车占比（0-1） | `order_cnt_21_09_7d_rate` |
| 睡眠不足天数 | `sleep_deprivation_days` |

空最小值表示无下限，空最大值表示无上限。

`是否组织化` 使用是/否选择，不使用最小值和最大值。

### 下载

下载当前筛选名单。

推荐 CSV，至少包含：

```text
司机id,综合画像
```

不要导出全量健康字段，不要导出全部原始字段。

### 建议策略

本次需要实现 `建议策略` 模块，放在 `综合画像` 之后、`司机资料` 之前。

建议策略用于帮助客服更有针对性地关怀司机，不是医学诊断，也不是事故因果判断。每条策略至少包含：

| 字段 | 含义 |
|---|---|
| `key` | 策略唯一 ID |
| `title` | 中文标题 |
| `evidence` | 命中依据 |
| `advice` | 建议内容 |
| `priority` | 展示排序 |

实现要求：

- 策略规则必须来自配置文件。
- 阈值数字和触发规则不能散落写死在 UI 渲染代码里。
- 当前生产策略只使用 `case_mean`：司机字段值高于死亡 case 对应字段的 case 均值才触发展示。
- 不使用非 case 均值口径的人工阈值、兜底阈值或宽泛城市/健康标签触发；后续阈值算法变更时再替换配置和计算层。
- 支持单指标规则，也支持 `any` 或 `all` 复合规则。
- 司机值为空、参考字段缺失、无有效样本或样本数不足时，不触发对应策略，并保留调试或说明信息。
- 没有命中任何策略时，展示 `常规关怀`。
- 策略标题和建议文案必须委婉、关怀、可解释、可操作，不使用医学诊断、事故因果判断或强制责备语气。

## 本次删减口径

和旧项目相比，本次不需要：

- `话术模板` 模块。
- 语料库拼接。
- 本地重建 SQLite 按钮。
- 写死在页面渲染代码里的建议策略。
- 持续累计历史司机明细的本地数据库或 CDN 发布包。

但不要因为删掉话术，就删掉查询、筛选、列表、详情、建议策略、下载这些核心功能。建议策略可以参考 `建议策略模块口径包_2026-07-06/`，但当前生产触发口径必须是：司机字段值高于死亡 case 对应字段的 case 均值才展示，并且保持配置驱动、计算层可替换。

## 数据来源建议

为了让新项目能独立跑通，建议先使用本地假数据或脱敏样例数据。

推荐数据流：

```text
sample-drivers.json / xlsx
  -> 构建脚本过滤最新 7 天
  -> 生成稳定静态版 drivers.json 日期索引
  -> 按自然日生成 daily/drivers-YYYY-MM-DD.json
  -> 前端按数据日期读取对应静态数据并筛选
  -> 点击司机后展示该日期文件中已加载静态档案
  -> 下载 CSV 只使用当前筛选结果的允许字段
```

后续接收每日新数据后，由于 CDN 一键部署包大小上限为 800MB，默认只使用上传当天向前含当天的最新 7 个自然日数据，并发布到 CDN 给页面读取。滚动窗口过滤应由构建或上传脚本完成，规则见 `docs/数据更新与CDN发布规则.md`。

如果另一个 agent 更适合后端实现，也可以：

```text
sample-drivers.xlsx / SQLite -> 后端 API -> 前端查询
```

但无论哪种实现，都必须满足相同功能和验收标准。
