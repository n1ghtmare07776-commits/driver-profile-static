# 近7天日均在线时长口径修正设计

## 背景

旧实现把当前数据窗口内同一司机已出现的 `server_dur_hour` 快照求平均，并展示为“七天平均时长”。风险司机库只包含命中入库条件的快照，不是完整的逐日出车流水；司机未入库的日期没有记录，因此该平均值不能代表七个自然日的真实工作时长。

已核对司机 `580545402486991` 的 2026-07-13 原始数据：当日服务时长为 `0`，近7日非预约单在线时长为 `110.7333`，近7日最小模型排名为 `1`。旧公式只看到一个当日服务快照，得到 `0 / 1 = 0`，计算过程没有错，但业务含义错误。

## 修正口径

- 原始字段：`lately_7d_except_sub_online_dur_hour`，中文为“近7日非预约单在线时长（小时）”。
- 派生字段：`avgOnlineDuration7d`。
- 公式：`avgOnlineDuration7d = lately_7d_except_sub_online_dur_hour / 7`。
- 分母固定为 7 个自然日，不再使用司机在风险库中的快照数。
- 原始字段缺失、空字符串或非有限数时，派生值为 `null`，页面不显示该短语。
- 原始字段为有效 `0` 时，派生值为 `0`，页面显示 `0h/日`。
- 页面最多显示两位小数并去掉末尾 `0`。
- 该指标描述在线时长，不得继续称为服务时长。

## 数据结构

每日紧凑文件新增原始字段 `lately_7d_except_sub_online_dur_hour`。组合筛选索引和司机直达索引新增派生字段 `avgOnlineDuration7d`，删除错误的 `avgServiceDuration7d` 与 `serviceDurationSampleDays`。

组合筛选索引的每个快照行独立按该行滚动七日字段计算。司机直达索引继续选择近七天最高排名对应的记录，并使用该记录自带的滚动七日字段计算派生值。排名、策略和日期选择逻辑保持不变。

## 前端展示

- 详情顶部：`近7天日均在线时长 15.82h/日`。
- 综合画像：`近7天日均在线时长15.82小时/日`。
- 不展示有效样本天数，因为该口径直接来自滚动七日源字段。
- 数据来源仍只展示原始模型排名命中日期、近七天最高排名和快照生成时间。

## 当前数据迁移

使用 2026-07-13 至 2026-07-17 的原始 Excel/ZIP，仅回填每日紧凑文件中的 `lately_7d_except_sub_online_dur_hour`，不改变司机行数、策略结果、排名或其他字段。回填完成后重建组合筛选索引、司机直达索引和 manifest 包体信息。

## 验收

- 脱敏测试中 `14 / 7 = 2`、有效 `0 / 7 = 0`、缺失值为 `null`。
- 司机 `580545402486991` 显示 `15.82h/日`，不再显示 `0h/日`。
- 完整司机 ID 查询和组合筛选详情口径一致。
- 现有排名、策略、进度条和 800MB 包体规则不回归。
