# 七天平均服务时长展示设计

## 目标

优化司机详情卡的服务时长口径：

- 删除数据来源栏中的“同步状态：静态快照”。
- 顶部副标题不再展示最高排名命中日的单日服务时长，改为当前窗口的“七天平均时长”。
- 综合画像中的“当日服务”改为“周平均服务时长”。

不改变近七天最高排名、建议策略、筛选条件、原始日文件和 CDN 800MB 口径。

## 口径

对每个司机，在 `manifest.actual_dates` / `drivers.json.dates` 对应的当前窗口内：

1. 收集该司机所有有效快照中的 `server_dur_hour`。
2. 只统计可解析为有限数字的值。
3. 平均值 = 有效服务时长之和 / 有效快照数。
4. 司机未出现的日期不按 0 小时计入，也不进入分母。
5. 有效值为 0 小时时，0 小时是有效样本，应进入分母。
6. 没有任何有效样本时，平均值为空，页面不展示该短语。

产物字段统一为：

- `avgServiceDuration7d`：七天窗口内有效快照的平均服务时长。
- `serviceDurationSampleDays`：参与平均的有效快照数，只用于验收和排查，V1 不在页面上显示。

## 方案

### 组合筛选索引

`scripts/build-filter-index.mjs` 仍完整读取当前窗口日文件。在构建快照记录时，同步按 `driverId` 累计有效 `server_dur_hour` 的总和与样本数。

全部日文件扫描完成后，为每条快照记录填入同一司机的 `avgServiceDuration7d` 和 `serviceDurationSampleDays`，然后写入 `filter-index.json.gz`。

这会增加两个紧凑数值字段，但不增加新网络请求，不需要用户点击后下载所有 daily 文件。字段会随现有 gzip 一起压缩。

`dist/filter-worker.js` 解码这两个字段，并把它们随最高排名命中快照一起返回。指定日期筛选时，仍展示整个当前窗口的平均值，不改成单日值。

### 司机 ID 直达索引

`scripts/build-direct-driver-lookup.py` 已在每个稳定哈希分桶中读取该司机的全部快照。在选出最高排名命中快照的同时，累计服务时长总和与有效样本数，并将两个聚合字段写入最终去重记录。

前端 `decodeDirectLookupPayload` 同步解码这两个字段。完整司机 ID 查询仍只下载一个 gzip 分桶。

### 页面展示

`dist/app.js` 和版本库前端源文件使用一个统一的平均时长格式化函数：

- 只接受有限数字。
- 四舍五入为两位小数。
- 移除小数末尾无意义的 0，例如 `3.50` 显示为 `3.5`。

顶部副标题：

```text
北京市 · 轻享专车 · 示例公司 · 41岁 · 七天平均时长 3.19h/日
```

综合画像：

```text
41岁，北京市，轻享专车，示例公司，连续出车1天，周平均服务时长3.19小时/日，……
```

数据来源栏从四项减为三项：

- 原始模型排名命中日期。
- 近七天最高排名。
- 快照生成时间。

不再渲染“同步状态”。

## 兼容与异常

- 当 `avgServiceDuration7d` 缺失或无效时，顶部和综合画像均跳过该短语，不回退展示单日值，避免口径混用。
- 单个司机只有一个有效快照时，平均值等于该快照的服务时长。
- 原始 `server_dur_hour` 继续保留在 daily 文件、筛选索引和内部指标中，高级筛选和策略计算仍使用原来的单日字段。
- 近七天最高排名命中逻辑不改。
- 不需要改写或重新编码 daily 文件。

## 性能和包体

- 页面不增加任何新请求。
- 司机 ID 直达仍只读取一个分片。
- 组合筛选仍由 Web Worker 处理。
- 新增两个数值字段可能小幅增加 `filter-index.json.gz` 和直达分片大小，但总包体仍必须低于 `800MB`。
- 验收时对比改造前后的 Worker 五次查询最慢耗时和最大直达分片，不允许突破现有测试阈值。

## 测试

### 聚合口径

- 同一司机有 3 个有效服务时长 `2`、`4`、`6` 时，平均值为 `4`，样本数为 `3`。
- 有效值 `0` 必须进入分母。
- 空值、`NaN`、无限值和不可解析文本不进入分子或分母。
- 司机缺少某个自然日快照时，不补 0。

### 产物契约

- `filter-index.json.gz` schema 包含 `avgServiceDuration7d` 和 `serviceDurationSampleDays`。
- `driver-lookup-index.json` 仍使用 `fnv1a32-modulo` 和 128 分桶。
- 直达分片的 schema 包含两个新字段。
- 近七天最高排名、快照日期和策略命中与改造前一致。

### 前端展示

- 顶部展示“七天平均时长”，不再显示模糊的“服务时长”。
- 综合画像显示“周平均服务时长”，不再显示“当日服务”。
- 数据来源栏不包含“同步状态”。
- 平均值显示最多两位小数，不显示 `NaN`、`undefined` 或空单位。

### 回归

- 运行导入脚本回归测试。
- 运行筛选索引、直达索引、进度条和 Worker 性能测试。
- 本地页面验证组合筛选和完整司机 ID 直达两条路径，两者显示相同的七天平均值。
- 验证页面错误日志为空，进度条不回退。

## 非目标

- 不新增七天平均服务时长筛选条件。
- 不修改建议策略的 `case_mean` 触发指标。
- 不把缺失日期当成 0 小时。
- 不新增页面提示、样本数标签或说明弹层。
