# 每日数据导入快速路径设计

## 目标

在不改变页面数据格式、组合筛选性能、司机 ID 直达性能、七天窗口规则和 800MB 安全机制的前提下，缩短每日新数据导入时间。

成功标准：

- 常规发布包未超过 800MB 时，Python 主导入流程只完整解码一次窗口内日文件。
- `drivers.json`、`filter-options.json`、`manifest.json`、`meta.json`、`filter-index.json.gz`、`driver-lookup-index.json` 和 `lookup/` 的对外契约保持不变。
- 页面筛选、近七天最高排名、完整司机 ID 查询和进度条行为保持不变。
- 真实数据副本上的每日导入耗时相较当前版本明显下降，目标进入 5-8 分钟范围；如果机器环境更快，以扫描次数和阶段耗时下降作为主要验收依据。

## 当前瓶颈

当前脚本在一次常规导入中多次完整读取窗口内日文件：

1. 汇总窗口司机数据。
2. 生成每日文件清单和行数。
3. 构建组合筛选索引。
4. 构建司机直达索引。
5. 执行 800MB 检查。
6. 两次最终 manifest/meta 回写时再次生成每日文件清单。

窗口内日文件目前约 576MB。Python 主流程中的重复读取和 JSON 解码没有改变最终产物，却增加了大量磁盘读取、对象构造和垃圾回收时间。

## 方案选择

采用保守的单次扫描快速路径。

不采用以下两项作为本次范围：

- 不并行运行组合筛选索引和司机直达索引构建，避免提高峰值内存和磁盘争用。
- 不实现增量筛选索引或增量司机去重，避免改变近七天最高排名的正确性边界。

## 架构

### 数据摘要

首次加载窗口内日文件时，同时生成一个内存摘要：

- `row_count`：窗口快照总数。
- `dates`：实际保留日期。
- `daily_files`：每个日期的路径和行数。
- `filter_options`：城市、公司、产品线和日期候选项。

摘要由已经解码的司机列表计算，不再为了行数或筛选项重新读取日文件。

### 静态索引写入

`write_static_indexes` 改为消费数据摘要：

- `drivers.json` 直接使用摘要中的 `daily_files`。
- `filter-options.json` 直接使用摘要中的 `filter_options`。
- `manifest.json` 和 `meta.json` 直接使用摘要中的 `row_count`、日期和每日文件清单。

最终包体数字稳定写入时，只重写这些小型元信息文件，不重新解析司机日文件。

### 800MB 快速路径

`enforce_size_limit` 接收当前摘要和日期：

- 如果整个 `dist/` 未超过 800MB，直接返回，不再次加载窗口司机数据。
- 如果超过上限，继续使用现有保守逻辑：从最旧日期开始剔除，重新加载剩余日文件、重建摘要和两套派生索引。

超限路径允许较慢，因为它不是正常的每日更新路径，且正确性优先。

### 派生索引

本次保持两套派生索引构建器不变：

- `build-filter-index.mjs` 继续生成 gzip 组合筛选索引。
- `build-direct-driver-lookup.py` 继续生成 128 个稳定哈希 gzip 分桶。

这样浏览器端文件格式、解压逻辑、Worker 查询和司机直达查询完全不受影响。

## 数据流

```text
读取新日 CSV
  -> 生成/覆盖新日期日文件
  -> 删除七天窗口外日文件
  -> 完整读取窗口日文件一次
  -> 生成 DatasetSummary
  -> 写 drivers/filter-options/manifest/meta
  -> 顺序重建 filter-index 和 driver-lookup
  -> 检查整个 dist/ 包体
      -> 未超限：复用 DatasetSummary，稳定写入最终元信息
      -> 超限：剔除最旧日期，重新加载并重建
```

## 错误处理

- 摘要日期必须与日文件名和司机 `dataDate` 一致；不一致时停止发布。
- 摘要行数必须等于 `daily_files.row_count` 之和。
- 派生索引构建失败时保持非零退出，不输出“导入成功”。
- 超过 800MB 时继续保留现有日期剔除和 `drop_reason` 记录逻辑。
- 最新一天单独仍超过 800MB 时继续停止发布，不强行上传。

## 测试

### 回归测试

- 给 `write_static_indexes` 传入摘要后，禁止调用日文件解码函数。
- 常规未超限路径调用 `enforce_size_limit` 时，禁止再次加载日文件。
- 摘要行数、每日行数之和、日期和筛选候选项保持一致。
- 超限测试继续验证最旧日期被剔除，且派生索引只包含剩余日期。

### 产物契约

- 继续运行现有组合筛选索引和司机直达索引契约测试。
- 对比优化前后的 manifest 字段、日文件清单和策略配置。
- 验证 `filter-index.json.gz` 行数等于窗口快照数。
- 验证司机直达分片最大文件仍低于 2MB。

### 真实数据基准

使用同一份发布目录副本和同一份 07-15 合并 CSV：

- 记录总耗时。
- 记录首次窗口加载、派生索引构建、包体检查和最终元信息写入阶段耗时。
- 验证页面北京筛选、完整司机 ID 查询和 07-15 快照详情。

## 非目标

- 不修改前端 UI。
- 不改变策略阈值和策略命中逻辑。
- 不改变近七天最高排名算法。
- 不压缩现有按日详情文件。
- 不减少司机、日期或页面必需字段。
- 不改变 CDN 上传目录和 deploy-id 规则。
