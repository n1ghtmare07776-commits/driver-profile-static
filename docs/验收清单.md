# 验收清单

另一个 agent 完成复刻后，必须逐项检查。

## 启动

- 能在本地启动网站。
- 能打开首页。
- 首页标题是 `司机画像`。
- 页面不是空白。
- 控制台没有主要 JS 报错。

## 页面外观

- 左侧是筛选栏，右侧是详情区。
- 背景是浅色/白色风格。
- 城市、公司、产品线下拉框默认收起。
- 点击城市输入框后才展开城市候选。
- 点击页面其他位置后下拉框可以收起。
- 文案全部为中文。

## 查询

- 输入一个存在的司机 ID，可以展示司机详情。
- 输入不存在的司机 ID，显示未找到。
- 输入非数字司机 ID，显示格式提示。
- 初始状态不展示默认全量司机列表。

## 多选筛选

- 城市支持搜索、多选、标签展示、右侧对勾。
- 公司支持搜索、多选、标签展示、右侧对勾。
- 产品线支持搜索、多选、标签展示、右侧对勾。
- 多选组内是 OR。
- 不同筛选组之间是 AND。

## 高级筛选

以下数值字段都支持最小值和最大值：

- 近七天最高排名
- 年龄
- 连续出车天数
- 服务时长
- 夜间出车占比（0-1）
- 睡眠不足天数

以下二选字段支持选择：

- 是否组织化（是/否）

必须验证：

- 只填最小值能筛选。
- 只填最大值能筛选。
- 同时填最小值和最大值能筛选。
- 填“近七天最高排名最大值 50000”时，只展示近 7 天命中过原始模型排名前 50000 的司机。
- 选择“是否组织化=是”能筛选。
- 选择“是否组织化=否”能筛选。
- 和城市/公司/产品线组合时仍是 AND。

## 下载

- 点击下载当前筛选名单，可以下载 CSV。
- CSV 至少有 `司机id` 和 `综合画像` 两列。
- CSV 内容只包含当前筛选结果。
- CSV 不应导出全量健康原始字段。

## 司机详情

- 详情页显示司机 ID。
- 显示城市、公司、产品线、年龄、数据日期。
- 详情页顶部展示简洁身份信息和 `综合风险`，不重复展示完整综合画像。
- 详情页来源信息和司机资料里展示 `近七天最高排名` 作为参考。
- 显示综合画像。
- 显示建议策略模块，位置在综合画像之后、司机资料之前。
- 显示司机资料分组卡片。
- 空值显示 `暂无数据`。
- 数字 0 显示为 `0`。

## 建议策略

- 策略规则来自配置文件，不是散落在页面渲染代码里。
- 每条规则有 `key`、`title`、`priority`、`threshold`、`evidence_template`、`advice_template`。
- 每条规则有 `translation.driver_script`、`translation.action_advice`、`translation.communication_tip`，页面主显话术转译内容。
- 当前生产阈值方法为 `case_mean`，大部分字段高于死亡 case 对应字段的 case 均值才触发展示；`min_sleep_duration` 等反向指标可配置为低于 case 均值触发。
- 建议策略按 P0/P1/P2/P3 优先级展示，高风险标签为红色，可解释标签为黄色。
- 对外话术不出现 `case均值`、`死亡case`、`阈值`、`内部均值` 或事故因果判断。
- 原始内部依据默认折叠在 `客服参考依据` 中。
- `复制话术` 只复制对外话术和可操作建议，不复制内部依据。
- `past_7_day_non_listening_period` 不再触发建议策略。
- 不使用非 case 均值口径的人工阈值、兜底阈值或宽泛城市/健康标签触发。
- 比较符可配置，当前生产规则使用 `>`。
- 能处理空值、非数值、字段缺失和样本不足。
- 字段缺失或无有效阈值时不触发对应策略，并保留原因。
- 支持 `any` 或 `all` 复合规则。
- 没有命中任何规则时展示 `常规关怀`。
- 文案委婉、礼貌、可解释，不做医学诊断，不做事故因果判断。
- 后续如需切换阈值算法，应只改配置和独立计算层，不修改页面渲染代码。

## 数据更新与 CDN

- 提供数据构建或上传脚本说明。
- CDN 一键部署包大小上限按 800MB 控制，发布前有包大小检查。
- 不持续追加历史数据库或历史司机明细到 CDN 发布包。
- 上传日为 2026-07-06 时，默认数据窗口是 2026-06-30 至 2026-07-06。
- 产物不包含默认窗口外的生产司机明细。
- manifest 记录 `generated_at`、`upload_date`、`window_start`、`window_end`、`row_count`。
- `generated_at` 由构建或上传脚本按运行时实时时间写入，不是手工固定日期。
- 页面左侧 `静态站点生成时间` 展示 `manifest.generated_at` 格式化后的时间，格式为 `YYYY/MM/DD HH:mm`。
- 页面展示当前数据窗口或最新快照日期。
- 筛选候选项、司机列表、详情、下载 CSV 和建议策略都基于当前 7 天窗口。
- 静态数据主索引为 `drivers.json`，只包含日期索引和每日文件清单，不包含多天司机明细。
- 每日司机明细文件为 `daily/drivers-YYYY-MM-DD.json`，只包含对应自然日页面需要的数据。
- `daily/drivers-YYYY-MM-DD.json` 不包含其他日期生产司机明细或页面不需要的全量原始字段。
- 导入新一天数据后，仍在 7 天窗口内的旧日期文件不会丢失。
- 点击司机或输入司机 ID 命中后，直接展示对应日期文件中已加载的静态档案，不再请求单司机详情 JSON。

## 明确不验收

- 不验收独立话术模板模块或回访任务系统。
- 不验收写死在页面渲染代码里的建议策略。
- 不验收 Hive/MySQL 接入。
- 不验收登录权限。
