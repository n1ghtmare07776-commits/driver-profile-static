# 用户流程

本文描述“司机画像”网站从首次部署、每日更新到日常查询的完整流程。更偏操作 SOP 的版本见 [每日更新说明](每日更新说明.md)。

## 1. 角色

| 角色 | 目标 |
|---|---|
| 使用网站的同事 | 查询司机画像、筛选司机名单、查看建议策略、下载 CSV |
| 维护数据的同事 | 拿到每日 Excel 后生成最近 7 天静态数据 |
| 公司内网 agent | 执行脚本、检查产物、递归上传 CDN 并返回链接 |

## 2. 首次部署流程

```text
收到项目包
  -> 解压并进入项目根目录
  -> 阅读 AGENTS.md 和 docs/
  -> 准备前端 dist/index.html、app.js、styles.css
  -> 导入第一份 Excel 生成 dist/data/
  -> 本地预览验收
  -> 递归上传整个 dist/
  -> 返回 CDN index.html 直链
```

### 2.1 输入

- 项目压缩包或项目文件夹。
- 第一份司机数据 Excel。
- 上传日期，优先从文件名中的 `YYYYMMDD` 推断。

### 2.2 操作

在项目根目录准备前端：

```bash
python3 scripts/prepare-static-site.py --out-dir dist
```

导入数据：

```bash
python3 scripts/import-driver-data.py <Excel文件路径> --upload-date YYYY-MM-DD --out-dir dist/data
```

本地预览：

```bash
python3 -m http.server 8000 --directory dist
```

部署 CDN：

```bash
python3 scripts/cdn-dir-deploy.py --dir dist --deploy-id driver-profile-YYYYMMDD-HHMM
```

### 2.3 输出

- 可访问的 CDN `index.html` 直链。
- 数据窗口。
- 司机数据条数。
- `dist/` 大小。
- 生成时间。
- deploy-id。

## 3. 每日更新流程

```text
拿到最新一天 Excel
  -> 交给 agent 或放入 data/incoming/
  -> 确认 upload_date
  -> 生成最近 7 天窗口
  -> 覆盖本次日期 daily/drivers-YYYY-MM-DD.json
  -> 保留窗口内已有日期文件
  -> 更新 drivers.json 日期索引、filter-options、manifest、strategy-thresholds
  -> 本地预览
  -> 使用新 deploy-id 递归上传整个 dist/
  -> 发出新的 CDN index.html 链接
```

### 3.1 单日 Excel 的处理口径

如果新 Excel 只包含最新一天数据，脚本会覆盖该日期的独立文件，并保留 `dist/data/daily/` 中仍在 7 天窗口内的旧日期文件。它不会把所有旧日期合并进 `drivers.json`，也不会因为只导入 7 月 7 日就删除仍在窗口内的 7 月 3 日。

如果当前文件夹里没有最近 6 天历史日期文件，且用户又只提供了当天 Excel，则无法凭空补齐 7 天窗口。此时 agent 应提示用户选择：

- 继续只发布当前可用日期。
- 补传最近 7 天 Excel 或合并文件后再发布。

### 3.2 推荐部署方式

每日更新推荐使用新的 `deploy-id` 递归上传整个 `dist/`。这样可以减少 CDN 或浏览器缓存导致的旧页面问题。

只有在公司内部明确要求沿用旧链接，并且页面代码没有变化时，才考虑只递归覆盖旧路径下的 `dist/data/`。

## 4. 网站查询流程

### 4.1 按司机 ID 查询

```text
打开网站
  -> 输入司机 ID
  -> 点击查询
  -> 加载当前日期或指定日期数据
  -> 左侧展示命中司机
  -> 右侧展示司机详情
```

异常状态：

- 司机 ID 格式不正确：页面提示格式问题。
- 没有找到司机：页面展示未找到状态。
- 数据文件读取失败：页面展示读取失败信息，引导检查 CDN 数据文件是否存在。

### 4.2 按筛选条件查询

```text
选择数据日期
  -> 选择城市/公司/产品线
  -> 填写高级筛选条件
  -> 点击查询
  -> 左侧展示所有匹配司机
  -> 点击某个司机
  -> 右侧展示该司机画像
```

筛选逻辑：

- 不同筛选组之间是 AND。
- 城市、公司、产品线组内是 OR。
- 空的最小值表示无下限。
- 空的最大值表示无上限。
- 是否组织化按是/否精确筛选。

## 5. 数据日期切换流程

```text
页面打开
  -> 读取 data/drivers.json 日期索引
  -> 读取 manifest 和筛选项
  -> 用户选择某个数据日期
  -> 前端读取 data/daily/drivers-YYYY-MM-DD.json
  -> 基于该日期数据执行筛选和详情展示
```

关键约束：

- `drivers.json` 不包含多天司机明细。
- 每个日期文件只包含该自然日司机明细。
- 日期切换必须加载对应日期文件，而不是继续使用上一次日期的数据。

## 6. 建议策略查看流程

```text
点击司机
  -> 前端读取该司机记录中的 strategies
  -> 先展示综合画像
  -> 再展示建议策略
  -> 最后展示司机资料
```

策略结果在数据导入阶段生成，前端只消费计算后的结果。当前生产口径为：

```text
司机字段值 > 死亡 case 对应字段 case 均值
```

如果没有任何策略命中，前端展示 `常规关怀`。

## 7. CSV 下载流程

```text
设置筛选条件
  -> 点击查询
  -> 确认左侧匹配结果
  -> 点击下载当前筛选名单
  -> 浏览器下载 CSV
```

CSV 只导出当前筛选结果，至少包含：

```text
司机id,综合画像
```

不得导出全量原始字段。

## 8. 交给 agent 的最佳流程

把项目包和 Excel 拖给公司内网 agent，然后使用 [每日更新说明](每日更新说明.md) 中的“一键更新部署提示词”。agent 应完成：

- 读取项目约束。
- 生成或更新 `dist/`。
- 检查按日独立数据文件。
- 本地预览。
- 递归上传 CDN。
- 返回 CDN `index.html` 直链和验收结果。
