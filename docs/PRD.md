# 司机画像 PRD

## 1. 背景

司机画像网站用于公司内部同事在联系司机前，快速查看司机的基础信息、出车状态、服务强度、疲劳相关指标和建议策略。当前版本是一个可部署到公司内网 CDN 的静态网站，重点解决三个问题：

- 能按司机 ID、城市、公司、产品线、数据日期和高级筛选条件找到目标司机。
- 能在同一页面查看司机综合画像、建议策略和资料详情。
- 能用每日 Excel 数据滚动生成最近 7 天静态数据，控制 CDN 发布包大小。

本项目不是营销页，也不是纯 UI 原型；必须是可以本地预览、可以上传 CDN、可以完成查询和下载的工具型页面。

## 2. 用户和场景

### 主要用户

- 公司内部需要查看司机画像的同事。
- 维护网站数据和部署 CDN 的同事或内网 agent。

### 典型场景

- 已知司机 ID：输入司机 ID，查看该司机最新或指定日期的画像。
- 批量筛选名单：按城市、公司、产品线、数据日期和高级筛选条件找出符合条件的司机。
- 查看建议策略：在综合画像后查看命中的关怀策略，理解触发依据和建议动作。
- 下载名单：导出当前筛选结果，用于后续跟进。
- 每日更新：拿到新一天 Excel 后，生成最近 7 天数据并部署新的 CDN 链接。

## 3. 目标

### 产品目标

- 复刻并保留当前“司机画像”最新版核心功能和清爽白色桌面布局。
- 增加并稳定保留建议策略模块。
- 支持最近 7 天按日独立查询，避免新一天数据导入后误删仍在窗口内的旧日期数据。
- 让同事可以通过 agent 使用一套提示词完成首次部署和每日更新。

### 工程目标

- 静态 CDN 版入口为 `dist/index.html`。
- 前端加载 `manifest.json`、`filter-options.json` 和 `drivers.json` 日期索引。
- 每个自然日司机明细单独保存在 `dist/data/daily/drivers-YYYY-MM-DD.json`。
- 部署时递归上传整个 `dist/`，不使用单 HTML 内联方案。
- 构建脚本写入实时 `manifest.generated_at`，页面展示为 `YYYY/MM/DD HH:mm`。

## 4. 非目标

- 不做话术模板。
- 不做语料库管理。
- 不做本地 SQLite 重建按钮。
- 不做登录、权限、审计。
- 不接 Hive、MySQL 或内部 API 后端。
- 不把页面改成深色原型、营销页或不可交互截图。
- 不把真实司机明细写进代码仓库或文档样例。

## 5. 功能需求

### 5.1 页面布局

- 页面标题必须是 `司机画像`。
- 左侧为筛选区和司机列表。
- 右侧为司机画像详情。
- 初始状态不展示全量司机列表。
- 页面语言为中文，整体为白色/浅色工具型界面。

### 5.2 查询和筛选

- 支持司机 ID 查询。
- 支持城市多选筛选。
- 支持公司多选筛选。
- 支持产品线（二级）多选筛选。
- 支持数据日期筛选。
- 支持高级筛选的最小值和最大值。
- 筛选组之间使用 AND。
- 城市、公司、产品线多选组内部使用 OR。
- 城市、公司、产品线下拉框默认收起，点击输入框后展开。
- 多选项支持搜索、点击加入、再次点击移除，并显示已选标签和右侧对勾。

高级筛选字段：

| 展示名 | 字段 |
|---|---|
| 年龄 | `age` |
| 连续出车天数 | `consecutive_days` |
| 服务时长 | `server_dur_hour` |
| 夜间出车占比（0-1） | `order_cnt_21_09_7d_rate` |
| 睡眠不足天数 | `sleep_deprivation_days` |
| 是否组织化 | `is_organized` |

`是否组织化` 使用是/否选择，不使用最小值和最大值。

### 5.3 司机列表

- 查询或筛选有结果时，左侧展示所有符合条件的司机列表。
- 点击列表中任意司机，右侧展示该司机详情和建议策略。
- 筛选结果优先按近七天最高排名升序排序；排名相同或缺失时，按疲劳分、风险分、数据日期和司机 ID 做稳定次级排序。
- 页面可以限制首屏显示数量，但必须显示真实匹配总数，不能把默认展示数量误写成匹配数量。

### 5.4 司机详情

司机详情至少包含：

- 司机 ID。
- 城市、公司、产品线。
- 年龄。
- 数据日期或快照日期。
- 综合画像。
- 建议策略。
- 司机资料分组。

模块顺序必须是：

```text
综合画像
建议策略
司机资料
```

空值显示 `暂无数据`，数字 0 必须显示为 `0`。

### 5.5 建议策略

建议策略用于帮助同事更有针对性地关怀司机，不做医学诊断，不做事故因果判断。

策略要求：

- 规则来自 `config/strategy-rules.json`。
- 页面渲染代码不写死阈值数字、触发规则和建议文案。
- 页面主显配置驱动的话术转译内容，不直接把 `case均值`、死亡 case、内部阈值等排查口径展示为对外话术。
- 当前生产触发口径只使用 `case_mean`。
- 大部分司机字段值高于死亡 case 文件中对应字段的 case 均值才触发展示；`min_sleep_duration` 等反向指标可以通过配置使用低于 case 均值触发。
- 不使用人工阈值、兜底阈值或宽泛城市/健康标签触发。
- 策略按 P0/P1/P2/P3 优先级展示；高风险标签显示为红色，可解释标签显示为黄色。
- `past_7_day_non_listening_period` 不再作为建议策略触发字段，只作为司机资料背景字段展示。
- 缺少司机指标、参考指标、有效样本或阈值时，不触发对应策略，并保留可排查说明。
- 无策略命中时展示 `常规关怀`。

每条策略至少包含：

| 字段 | 含义 |
|---|---|
| `key` | 策略唯一 ID |
| `title` | 中文标题 |
| `evidence` | 命中依据 |
| `advice` | 建议内容 |
| `priority` | 展示排序 |
| `priority_tier` | P0/P1/P2/P3 业务优先级 |
| `badges` | 高风险、可解释等展示标签 |
| `translation.driver_script` | 对外话术，客服可直接念给司机 |
| `translation.action_advice` | 可操作建议 |
| `translation.communication_tip` | 沟通提醒 |

页面展示要求：

- 策略卡主显 `对外话术`、`可操作建议` 和 `沟通提醒`。
- `客服参考依据` 默认折叠展示原内部依据和内部建议。
- `复制话术` 只复制对外话术和可操作建议，不复制内部依据。
- 对外话术必须委婉、关怀、可解释、可操作，不做医学诊断，不做事故因果判断。

完整策略排序和标签口径见 [建议策略优先级与标签口径](建议策略优先级与标签口径.md)。

### 5.6 CSV 下载

- 支持下载当前筛选名单 CSV。
- CSV 至少包含 `司机id` 和 `综合画像` 两列。
- CSV 只包含当前筛选结果。
- 不导出全量原始字段或页面不需要的敏感明细。

## 6. 数据需求

### 6.1 7 天窗口

由于 CDN 一键部署包大小上限为 800MB，生产发布包默认只保留上传日向前含当天的最近 7 个自然日数据。

示例：

```text
upload_date = 2026-07-07
window_start = 2026-07-01
window_end = 2026-07-07
```

筛选候选项、司机列表、详情、CSV 下载和建议策略结果都必须基于当前 7 天窗口。

如果最近 7 天静态产物超过 CDN 800MB 上限，系统必须优先裁剪非页面必需字段和使用紧凑 JSON。仍超过时，允许从最旧日期开始剔除，只保留 800MB 内可容纳的最近日期。实际保留日期和剔除原因必须写入 manifest。

### 6.2 按日独立数据

CDN 静态版必须使用按日独立数据文件：

```text
dist/data/drivers.json
dist/data/daily/drivers-YYYY-MM-DD.json
dist/data/filter-options.json
dist/data/strategy-thresholds.json
dist/data/manifest.json
```

其中 `drivers.json` 只作为日期索引和每日文件清单，不允许合并多天司机明细。每天司机明细必须写入独立文件。导入新一天数据时，脚本只能覆盖本次输入对应日期文件，并保留仍在 7 天窗口内的旧日期文件。

### 6.3 元信息

`manifest.json` 至少记录：

- `generated_at`
- `upload_date`
- `window_start`
- `window_end`
- `row_count`
- `contains_queryable_driver_data`
- `data_mode`
- `primary_data_file`
- `daily_files`
- `actual_dates`
- `dropped_dates`
- `drop_reason`
- `package_size_mb`
- `package_size_limit_mb`

`generated_at` 必须由构建或上传脚本运行时写入，不能手工固定。

## 7. 验收标准

- 本地能启动并打开 `index.html`。
- 页面标题是 `司机画像`。
- 城市、公司、产品线多选可搜索、可勾选、可移除。
- 数据日期切换能读取对应 `daily/drivers-YYYY-MM-DD.json`。
- 输入存在的司机 ID 能展示详情。
- 城市筛选后左侧能展示司机列表，点击司机能切换右侧详情。
- 建议策略模块位于综合画像之后、司机资料之前。
- 无策略命中时显示 `常规关怀`。
- 下载 CSV 可用，且只导出允许字段。
- 页面左侧展示 `静态站点生成时间`，时间来自 `manifest.generated_at`。
- 导入新一天数据后，仍在 7 天窗口内的旧日期文件不丢失。
- 发布包大小低于 800MB。
- CDN 链接可访问 `index.html`、`data/manifest.json`、`data/drivers.json`、`data/filter-options.json` 和对应日期的 `data/daily/drivers-YYYY-MM-DD.json`。

## 8. 暂未纳入当前版本的决策

以下问题可以后续再决策，不阻塞当前静态版：

- 是否需要登录、权限和访问审计。
- 是否从每日 Excel 改为公司内部 API 或数仓自动同步。
- 同一司机在 7 天内有多条记录时，列表是否默认展示所有日期快照，还是只展示最新快照。
- 是否需要把 CDN 发布从 agent 脚本升级为固定流水线。
- 是否需要为不同团队配置不同筛选默认项或导出字段。
