# 数据字典

本文定义“司机画像”静态站点使用的数据字段、静态 JSON 文件结构和导出字段。真实司机数据不得写入文档样例，以下示例均为脱敏示例。

## 1. 源数据字段

每日 Excel/CSV/JSON 输入至少需要包含司机 ID、数据日期和页面筛选/展示所需字段。脚本会把部分中文列名映射为标准字段。

| 标准字段 | 常见中文列名 | 含义 | 类型 | 是否必需 |
|---|---|---|---|---|
| `driver_id` | `司机id` | 司机 ID | string | 是 |
| `dt` | `dt` | 数据日期 | date string | 是 |
| `resident_city_name` | `常驻城市` | 城市 | string | 建议 |
| `company_name` | `公司名称` | 公司 | string | 建议 |
| `product_level2_name` | `业务线` | 产品线二级名称 | string | 建议 |
| `age` | `年龄` | 年龄 | number | 建议 |
| `consecutive_days` | `连续出车天数` | 连续出车天数 | number | 建议 |
| `server_dur_hour` | `当日服务时长（小时）` | 当日服务时长 | number | 建议 |
| `server_dur_sum_30d` | `近30日服务时长（小时）` | 近 30 日服务时长 | number | 可选 |
| `order_cnt_21_09_7d_rate` | `近7天21-9点完单量占比` | 夜间出车占比 | number | 建议 |
| `sleep_deprivation_days` | `近7天睡眠不足天数` | 睡眠不足天数 | number | 建议 |
| `is_organized` | `是否组织化` | 是否组织化 | string/bool | 建议 |
| `risk_tier_rank` | `当日模型排名` / `司机分层排名` | 源表模型排名，可能来自全量模型或上游排名空间，不等同于静态站点当前窗口人数 | number | 可选 |
| `risk_tier_score` | `司机库分层分数` | 风险分 | number | 可选 |
| `tired_score` | `劳累分场景分数` | 疲劳分 | number | 可选 |

说明：

- `dt` 是数据日期功能的基础字段。没有可解析日期时，数据日期筛选会失去意义。
- 当前 UI 不再把“当日在线时长”和“历史最长连续出车”作为高级筛选或独立建议策略入口。
- 脚本可以保留部分兼容字段，但 CDN 输出应只保留页面需要的字段。

## 2. 标准化司机字段

导入脚本会把源字段清洗成前端使用的字段。

| 前端字段 | 含义 | 来源 |
|---|---|---|
| `driverId` | 司机 ID | `driver_id` |
| `city` | 城市 | `resident_city_name` |
| `company` | 公司 | `company_name` |
| `product` | 产品线 | `product_level2_name` |
| `age` | 年龄 | `age` |
| `consecutive_days` | 连续出车天数 | `consecutive_days` |
| `server_dur_hour` | 当日服务时长 | `server_dur_hour` |
| `server_dur_sum_30d` | 近 30 日服务时长 | `server_dur_sum_30d` |
| `order_cnt_21_09_7d_rate` | 夜间出车占比 | `order_cnt_21_09_7d_rate` |
| `sleep_deprivation_days` | 睡眠不足天数 | `sleep_deprivation_days` |
| `isOrganized` | 是否组织化 | `is_organized` |
| `riskTierRank` | 原始模型排名 | `risk_tier_rank` |
| `bestRiskTierRank` | 近 7 天内该司机命中过的最佳原始模型排名 | 前端按司机 ID 去重时计算 |
| `bestWindowRank` | 近 7 天静态站点窗口内去重后的站内排名，从 1 开始连续编号 | 前端按 `bestRiskTierRank` 排序后计算 |
| `riskTierScore` | 风险分 | `risk_tier_score` |
| `tiredScore` | 疲劳分 | `tired_score` |
| `dataDate` | 数据日期 | `dt` |
| `subtitle` | 列表副标题 | 字段组合生成 |
| `summary` | 综合画像 | 字段组合生成 |
| `strategies` | 建议策略 | 策略计算层生成 |

说明：

- 页面顶部、司机列表和高级筛选使用 `bestWindowRank`，它代表当前 CDN 静态数据窗口内的站内相对顺序。
- `riskTierRank` / `bestRiskTierRank` 只作为“原始模型排名”展示和排查参考，可能大于当前静态站点可查询司机数。

## 3. 静态产物总览

当前 CDN 静态版使用多文件结构：

```text
dist/data/
├── drivers.json
├── daily/
│   ├── drivers-YYYY-MM-DD.json
│   └── drivers-YYYY-MM-DD.json
├── filter-options.json
├── meta.json
├── manifest.json
└── strategy-thresholds.json
```

注意：静态 JSON 包含可查询司机数据，因此只能进入受控 CDN 发布产物，不能作为文档样例或代码仓库里的真实数据提交。

## 4. `drivers.json`

`drivers.json` 是最近 7 天窗口内的日期索引，不承载司机明细。

示例：

```json
{
  "mode": "daily-static-index",
  "latestDate": "2026-07-07",
  "dates": ["2026-07-03", "2026-07-07"],
  "files": [
    {
      "date": "2026-07-03",
      "path": "daily/drivers-2026-07-03.json",
      "row_count": 200000
    },
    {
      "date": "2026-07-07",
      "path": "daily/drivers-2026-07-07.json",
      "row_count": 165000
    }
  ]
}
```

| 字段 | 类型 | 含义 |
|---|---|---|
| `mode` | string | 固定为 `daily-static-index` |
| `latestDate` | string | 当前窗口内最新数据日期 |
| `dates` | string[] | 当前窗口内可查询日期 |
| `files` | array | 每个日期对应的独立明细文件 |

## 5. `daily/drivers-YYYY-MM-DD.json`

每个日期文件只包含对应自然日的司机明细。

示例：

```json
{
  "mode": "daily-static",
  "date": "2026-07-07",
  "drivers": [
    {
      "driverId": "100000000001",
      "city": "北京市",
      "company": "示例公司A",
      "product": "快车",
      "age": 56,
      "consecutive_days": 5,
      "server_dur_hour": 9.5,
      "server_dur_sum_30d": 180,
      "order_cnt_21_09_7d_rate": 0.42,
      "sleep_deprivation_days": 2,
      "isOrganized": "是",
      "riskTierRank": 128,
      "riskTierScore": 82.1,
      "tiredScore": 76.3,
      "dataDate": "2026-07-07",
      "subtitle": "北京市 · 快车 · 示例公司A",
      "summary": "56岁，北京市，快车，连续出车5天，当日服务9.5小时。",
      "strategies": [
        {
          "key": "regular-care",
          "title": "常规关怀",
          "evidence": "当前没有任何指标高于case均值。",
          "advice": "可做常规状态确认，提醒司机保持安全驾驶、合理休息和规律作息。",
          "priority": 999
        }
      ]
    }
  ]
}
```

约束：

- 不包含其他日期司机明细。
- 不包含页面不需要的全量原始字段。
- 点击司机时，前端直接使用该记录组装详情。
- 不再为每个司机请求单独详情 JSON。

## 6. `filter-options.json`

用于前端渲染筛选候选项。

示例：

```json
{
  "cities": ["北京市", "上海市"],
  "companies": ["示例公司A", "示例公司B"],
  "products": ["快车", "专车"],
  "dates": ["2026-07-03", "2026-07-07"]
}
```

候选项必须来自当前 7 天窗口内可查询数据。

## 7. `manifest.json`

记录本次静态数据生成和发布相关信息。

示例：

```json
{
  "generated_at": "2026-07-07T08:20:00+00:00",
  "upload_date": "2026-07-07",
  "window_start": "2026-07-01",
  "window_end": "2026-07-07",
  "row_count": 365000,
  "contains_queryable_driver_data": true,
  "data_mode": "daily_static",
  "primary_data_file": "drivers.json",
  "daily_files": [
    {
      "date": "2026-07-07",
      "path": "daily/drivers-2026-07-07.json",
      "row_count": 165000
    }
  ],
  "package_size_limit_mb": 800
}
```

| 字段 | 含义 |
|---|---|
| `generated_at` | 脚本运行时写入的生成时间 |
| `upload_date` | 上传日，也是 7 天窗口锚点 |
| `window_start` | 7 天窗口开始日期 |
| `window_end` | 7 天窗口结束日期 |
| `row_count` | 当前窗口内司机记录总数 |
| `contains_queryable_driver_data` | 标记静态 JSON 是否包含可查询司机数据 |
| `data_mode` | 当前为 `daily_static` |
| `primary_data_file` | 当前为 `drivers.json` |
| `daily_files` | 当前窗口内每日明细文件 |
| `package_size_limit_mb` | CDN 发布包大小上限 |

页面左侧 `静态站点生成时间` 读取 `manifest.generated_at` 并格式化为 `YYYY/MM/DD HH:mm`。

## 8. `strategy-thresholds.json`

记录建议策略阈值和排查信息。

示例：

```json
{
  "generatedAt": "2026-07-07T08:20:00+00:00",
  "policy": "case_mean",
  "rules": [
    {
      "key": "service-duration-high",
      "enabled": true,
      "thresholdPolicy": "case_mean",
      "operator": ">",
      "thresholdValue": 8.6,
      "sampleCount": 120
    }
  ]
}
```

当前生产口径只使用 `case_mean`：大部分字段高于死亡 case 对应字段的 case 均值才触发；`min_sleep_duration` 等反向指标可配置为低于 case 均值触发。

## 9. 建议策略配置字段

配置文件为 `config/strategy-rules.json`。

| 字段 | 含义 |
|---|---|
| `version` | 规则版本 |
| `thresholdDefaults.policy` | 默认阈值方法，当前为 `case_mean` |
| `rules[].key` | 策略唯一 ID |
| `rules[].enabled` | 是否启用 |
| `rules[].title` | 策略标题 |
| `rules[].category` | 策略分类 |
| `rules[].priority` | 展示顺序 |
| `rules[].priority_tier` | 业务优先级，当前使用 P0/P1/P2/P3 |
| `rules[].badges` | 展示标签配置，例如高风险、可解释 |
| `rules[].driver_metric` | 司机指标字段 |
| `rules[].condition` | 复合条件，支持 `any` / `all` |
| `rules[].threshold.reference_metric` | case 参考字段 |
| `rules[].threshold.operator` | 比较符，大部分字段使用 `>`；`min_sleep_duration` 等反向指标可配置为 `<` |
| `rules[].evidence_template` | 命中依据模板 |
| `rules[].advice_template` | 建议文案模板 |
| `fallbackRule` | 无策略命中时的常规关怀 |

建议策略的排序、标签和字段优先级见 [建议策略优先级与标签口径](建议策略优先级与标签口径.md)。

## 10. CSV 导出字段

CSV 至少包含：

| 列名 | 含义 |
|---|---|
| `司机id` | 司机 ID |
| `综合画像` | 页面展示的综合画像摘要 |

可以增加低敏展示字段，例如数据日期、城市、公司、产品线，但不得导出全量原始字段。
