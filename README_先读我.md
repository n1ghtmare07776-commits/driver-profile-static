# 司机画像复刻需求包

这个文件夹用于让另一个 agent 在新文件夹里独立复刻“司机画像”项目的最新版核心功能和外观，也可以作为日常更新和 CDN 部署的维护包使用。

如果只是复刻需求，不要求沿用旧项目代码；另一个 agent 可以用自己熟悉的技术栈实现，但必须满足本文档中的产品功能、页面外观、数据契约和验收标准。如果是日常更新，请优先阅读 `docs/每日更新说明.md`，把最新 Excel 拖给公司内网 agent 后按提示词一键生成并部署 CDN。

## 复刻目标

做一个新的“司机画像”网站，同事可以：

- 输入司机 ID 查询司机画像。
- 使用城市、公司、产品线、日期组合筛选司机。
- 使用高级数值筛选。
- 查看司机档案页面。
- 查看建议策略。
- 下载当前筛选名单 CSV。

如果需要按产品经理规范交接，请优先阅读：

- `docs/PRD.md`：产品背景、目标、范围、需求和验收标准。
- `docs/用户流程.md`：首次部署、每日更新、查询筛选、日期切换和下载流程。
- `docs/数据字典.md`：源字段、标准字段、静态 JSON 结构和导出字段。
- `docs/技术方案.md`：静态站点架构、按日独立数据、脚本流程和 CDN 部署方案。

## 本次明确不需要

- 不需要话术模板。
- 需要建议策略，但必须是配置驱动、阈值可替换的策略模块。
- 不需要写死在页面渲染代码里的策略判断或阈值数字。
- 不需要语料库。
- 不需要登录、权限、审计。
- 不需要接 Hive、MySQL 或公司内部部署系统。

## 重要提醒

不要让另一个 agent 简化成只有一个输入框的页面。必须保留多选筛选、高级筛选、查询结果列表、画像展示和下载能力。

由于 CDN 一键部署包大小上限为 800MB，后续每日新增数据时，默认只保留上传当天向前含当天的最新 7 个自然日数据，不持续累计历史数据库或历史司机明细。具体规则见 `docs/数据更新与CDN发布规则.md`。

每日数据可以通过脚本生成 CDN 静态产物：

```text
python3 scripts/import-driver-data.py data/incoming/司机库数据_20260706.xlsx --upload-date 2026-07-06 --out-dir dist/data
```

生成后的 `dist/data/drivers.json` 是日期索引；每一天司机明细会保存在 `dist/data/daily/drivers-YYYY-MM-DD.json`。导入新一天数据时，脚本会保留 7 天窗口内已有日期文件，不会因为只导入最新一天而覆盖掉旧日期。

给负责维护的同事使用的完整 SOP 见 `docs/每日更新说明.md`。最推荐的部署方式是让公司内网 agent 使用新的 `deploy-id` 递归上传整个 `dist/`，并返回 CDN `index.html` 直链，例如：

```text
python3 scripts/cdn-dir-deploy.py --dir dist --deploy-id driver-profile-YYYYMMDD-HHMM
```

不要让另一个 agent 直接改旧交接包。建议它在一个全新的项目文件夹里实现，然后按 `docs/验收清单.md` 验收。
