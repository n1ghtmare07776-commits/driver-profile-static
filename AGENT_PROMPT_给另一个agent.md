请在一个全新的项目文件夹中复刻“司机画像”网站的最新版核心功能和外观。

## 任务目标

实现一个可本地运行的网站，给同事查询司机画像使用。用户可以输入司机 ID，也可以通过城市、公司、产品线、日期和高级筛选组合筛选司机名单，点击名单中的司机查看详情，查看建议策略，并下载当前筛选结果 CSV。

## 必须实现

1. 页面标题：`司机画像`。
2. 左侧筛选栏，右侧详情区。
3. 司机 ID 输入框。
4. 城市多选搜索框。
5. 公司多选搜索框。
6. 产品线（二级）多选搜索框。
7. 数据日期下拉框。
8. 高级筛选，支持以下字段：
   数值字段支持最小值和最大值：
   - 年龄
   - 连续出车天数
   - 服务时长
   - 夜间出车占比（0-1）
   - 睡眠不足天数
   二选字段支持选择：
   - 是否组织化（是/否）
9. 筛选语义：
   - 不同筛选组之间是 AND。
   - 城市、公司、产品线组内多选是 OR。
   - 空的最大值代表无上限，空的最小值代表无下限。
10. 查询结果列表。
11. 司机详情页，至少包含：
    - 司机 ID
    - 城市
    - 公司
    - 产品线
    - 年龄
    - 数据日期
    - 综合画像
    - 建议策略
    - 司机资料分组
12. 建议策略模块：
    - 放在 `综合画像` 之后、`司机资料` 之前。
    - 策略规则来自配置文件。
    - 阈值计算和触发逻辑在独立计算层，不写死在页面渲染函数里。
    - 当前生产策略只使用 `case_mean`：大部分字段高于死亡 case 对应字段的 case 均值才触发展示；`min_sleep_duration` 等反向指标可通过配置使用低于 case 均值触发。
    - 不要使用非 case 均值口径的人工阈值、兜底阈值或宽泛城市/健康标签触发；后续阈值算法变更时再替换配置和计算层。
    - 支持 `any` 或 `all` 复合规则。
    - 缺字段、无有效样本或无阈值时不触发，并保留说明。
    - 无命中时展示 `常规关怀`。
    - 文案委婉、关怀、可解释，不做医学诊断或事故因果判断。
13. 每日数据/CDN：
    - 由于 CDN 一键部署包大小上限为 800MB，不能持续追加历史数据库或历史司机明细。
    - 后续每次给新一天数据时，默认只保留上传当天向前含当天的最新 7 个自然日数据。
    - 构建或上传脚本负责过滤 7 天窗口、生成按日独立静态 JSON、策略阈值文件和 manifest。
    - CDN 静态版采用按日独立的稳定静态版方案：首页加载 `manifest.json`、`filter-options.json` 和 `drivers.json` 日期索引，选择数据日期后读取 `daily/drivers-YYYY-MM-DD.json`。
    - `drivers.json` 只能包含日期索引和每日文件清单，不允许包含多天司机明细。
    - 每个 `daily/drivers-YYYY-MM-DD.json` 只能包含对应自然日页面查询、筛选、列表、详情、综合画像和建议策略展示所需字段，不放其他日期生产司机明细或页面不需要的全量原始字段。
    - 每次导入新一天数据时，必须保留 7 天窗口内已有日期文件，不能因为导入 07-07 单日数据而删除 07-03 等仍在窗口内的数据。
    - manifest 记录 `generated_at`、`upload_date`、`window_start`、`window_end`、`row_count`。
    - CDN 部署必须使用目录递归上传，入口是 `dist/index.html`，不能只上传单个 HTML、`index-inline.html` 或 `widget.json`。
    - 推荐使用新的 `deploy-id` 上传整个 `dist/`，例如 `driver-profile-YYYYMMDD-HHMM`，最后返回 CDN `index.html` 直链。
14. 首页加载静态数据时必须显示进度条。
15. 下载当前筛选名单 CSV，至少包含：
    - `司机id`
    - `综合画像`

## 不需要实现

- 不需要话术模板。
- 需要建议策略模块。
- 不需要写死在页面渲染代码里的建议策略。
- 不需要客服话术语料库。
- 不需要登录权限。
- 不需要接 Hive 或 MySQL。

## 外观要求

请复刻当前白色版本的视觉方向：

- 左侧 300px 左右筛选栏。
- 右侧大面积详情区。
- 白色卡片、浅灰背景、蓝色主色。
- 卡片圆角约 6-8px。
- 详情页包含顶部司机卡片、综合画像、建议策略、司机资料卡片。
- 不要做深色主题。
- 不要做营销页。

## 验收方式

完成后请按 `docs/验收清单.md` 和 `docs/数据更新与CDN发布规则.md` 逐项验证，并反馈：

- 本地启动地址。
- 使用的样例数据条数。
- 查询一个存在司机 ID 的结果。
- 城市/公司/产品线多选是否可用。
- 高级筛选是否可用。
- 是否组织化筛选是否可用。
- 建议策略配置文件路径、阈值计算方法和示例命中结果。
- 数据构建/上传脚本如何默认截取最新 7 天。
- 一键导入脚本命令，例如 `python3 scripts/import-driver-data.py data/incoming/司机库数据_20260706.xlsx --upload-date 2026-07-06 --out-dir dist/data`。
- 一键 CDN 目录部署命令，例如 `python3 scripts/cdn-dir-deploy.py --dir dist --deploy-id driver-profile-YYYYMMDD-HHMM`。
- 首页是否加载稳定静态版 `drivers.json` 日期索引。
- 选择某个数据日期后，是否加载对应的 `daily/drivers-YYYY-MM-DD.json`。
- 导入新一天数据后，仍在 7 天窗口内的旧日期文件是否保留。
- 加载每日司机数据时是否显示进度条。
- CSV 下载是否只有允许字段。
